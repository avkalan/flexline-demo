Curve Insight · Curve Flex BNPL engine
Curve Flex is the strategic centrepiece of the acquisition. It handles instalment scheduling, APR calculation, plan management, and purchase protection — all as a white-label API.
Technical note: the original transaction is NOT reversed. Curve creates a virtual funding source, reallocates the charge to the FlexLine credit line, and the Mastercard network records the change. The merchant sees nothing. This is Curve's core patent-protected capability — POST /flex/convert is the single most valuable API call in the entire integration.

Curve Insight · Virtual Mastercard issuance
The bank does NOT need a card BIN sponsor relationship. Curve is a Mastercard principal member — card issuance is a managed service. This removes 6–12 months of bank onboarding overhead.
Sensitive card data (PAN, CVV) is never stored by the bank. Curve's security layer returns tokenised values only after biometric re-authentication is confirmed. Apple Pay and Google Pay provisioning uses Curve's existing wallet tokenisation pipeline — no separate TSP agreements needed.

Curve Insight · Consolidated payment architecture
Curve's schedule API aggregates all active Flex plans into a single monthly total — reducing default risk (one missed payment vs. many) and customer complexity.
The bank owns the direct debit mandate; Curve provides the schedule aggregation. The customer can change their payment date once per cycle (days 1–28). Curve enforces the constraint and returns the next valid collection date. Webhook flex.payment.failed triggers retry logic managed by the bank.
